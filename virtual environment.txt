1️⃣ What Is a Virtual Environment?

A virtual environment (venv) is:

An isolated Python environment

With its own Python interpreter

Its own site-packages

Independent of system Python & other projects

👉 Each project can have different package versions without conflict.

2️⃣ The Problem Virtual Environments Solve
❌ Without Virtual Environments
pip install django==2.2
pip install django==4.2


💥 Conflict — only one version can exist globally.

✅ With Virtual Environments
Project A → Django 2.2
Project B → Django 4.2


✔ No conflict
✔ Clean system Python
✔ Reproducible builds

3️⃣ What Exactly Is Isolated?

Inside a virtual environment:

✔ Python executable
✔ pip
✔ site-packages directory
✔ Scripts/binaries

❌ OS libraries
❌ Kernel
❌ CPU / RAM isolation (this is NOT Docker)

4️⃣ How Virtual Environments Work (Internals)

When you activate a venv:

python points to:

venv/bin/python   (Linux/Mac)
venv\Scripts\python.exe  (Windows)


pip installs packages only inside that folder

PATH environment variable is modified

5️⃣ Creating Virtual Environments (Standard Way)
Using built-in venv (Recommended)
python -m venv venv


Creates:

venv/
├── bin/ or Scripts/
├── lib/
├── pyvenv.cfg

6️⃣ Activating a Virtual Environment
🔹 Windows
venv\Scripts\activate

🔹 Linux / macOS
source venv/bin/activate


After activation:

(venv) $

7️⃣ Deactivating the Virtual Environment
deactivate

8️⃣ Installing Packages Inside a venv
pip install requests


Check location:

pip show requests


✔ Installed only inside venv

9️⃣ Verifying Isolation
which python     # Linux/Mac
where python     # Windows


Output points to:

venv/bin/python

🔟 Using Virtual Environment in Python Code

Your code doesn’t change:

import requests

r = requests.get("https://example.com")
print(r.status_code)


The environment decides where imports come from.

1️⃣1️⃣ requirements.txt (VERY IMPORTANT)
Freeze dependencies
pip freeze > requirements.txt


Example:

requests==2.31.0
flask==3.0.0

Install dependencies
pip install -r requirements.txt


✔ Reproducible environment
✔ Used in CI/CD, AWS, Docker

1️⃣2️⃣ Virtual Environments for Different Python Versions
python3.11 -m venv venv


or

py -3.10 -m venv venv   # Windows

1️⃣3️⃣ Common Virtual Environment Tools
1️⃣ Built-in venv (Most common)

✔ Simple
✔ Lightweight
❌ No dependency resolution

2️⃣ virtualenv
pip install virtualenv
virtualenv venv


✔ Faster
✔ Works with older Python

3️⃣ pipenv (Env + dependency manager)
pip install pipenv
pipenv install requests
pipenv shell


Files:

Pipfile
Pipfile.lock

4️⃣ poetry (Modern & powerful)
pip install poetry
poetry init
poetry install
poetry shell


Files:

pyproject.toml
poetry.lock

5️⃣ conda (Data Science)
conda create -n myenv python=3.11
conda activate myenv


✔ Manages Python + native libs

1️⃣4️⃣ Virtual Environment vs Docker
Virtual Env	Docker
Python-level	OS-level
Lightweight	Heavy
Same OS	Any OS
Dev focused	Deployment focused

➡ Often used together

1️⃣5️⃣ Best Practices ⭐

✔ One venv per project
✔ Never commit venv/ folder
✔ Always commit requirements.txt
✔ Name it .venv or venv
✔ Use .gitignore

Example .gitignore:

venv/
__pycache__/

1️⃣6️⃣ Common Mistakes

❌ Installing packages globally
❌ Forgetting to activate venv
❌ Mixing conda + pip blindly
❌ Committing venv directory
